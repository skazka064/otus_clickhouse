# Опишите две или более топологий объединения экземпляров в шарды, указав фактор репликации и количество шардов.

###  Топология 1: Один шард, несколько реплик

все данные хранятся в одном логическом шарде, но полностью дублируются на нескольких серверах (репликах). Это классическая конфигурация для повышения отказоустойчивости и распределения нагрузки при чтении
Количество шардов =1 . Фактор репликации например 2 или 3

```
<remote_servers>
    <cluster> 
        <shard>
            <internal_replication>true</internal_replication>
            <replica>
                <host>clickhouse-01</host>
                <port>9000</port>
            </replica>
            <replica>
                <host>clickhouse-02</host>
                <port>9000</port>
            </replica>
        </shard>
    </cluster>
</remote_servers>
```

### Топология 2: Несколько шардов, несколько реплик 2-2
Данные разделяются на части (шарды) и распределяются по серверам. Каждая часть данных (каждый шард) имеет свои собственные реплики для отказоустойчивости
Количество шардов, например два и фактор репликации два

```
<remote_servers>
    <cluster>
        <shard>
            <internal_replication>true</internal_replication>
            <replica>
                <host>clickhouse-01</host>
                <port>9000</port>
            </replica>
            <replica>
                <host>clickhouse-03</host>
                <port>9000</port>
            </replica>
        </shard>
        <shard>
            <internal_replication>true</internal_replication>
            <replica>
                <host>clickhouse-02</host>
                <port>9000</port>
            </replica>
            <replica>
                <host>clickhouse-04</host>
                <port>9000</port>
            </replica>
        </shard>
    </cluster>
</remote_servers>

```

### Я объединил четыре хоста, сделав два шарда по две реплики. Конфиги приложил ниже.

### Первый хост

```

    <listen_host>0.0.0.0</listen_host>
    <listen_try>1</listen_try>

    <remote_servers>
        <cluster_3_replicas>
            <shard>
                <internal_replication>true</internal_replication>
                <replica>
                    <host>clickhouse-01</host>
                    <port>9000</port>
                </replica>
                <replica>
                    <host>clickhouse-02</host>
                    <port>9000</port>
                </replica>
            </shard> 
            <shard>
                <internal_replication>true</internal_replication>
                <replica>
                    <host>clickhouse-03</host>
                    <port>9000</port>
                </replica>
                <replica>
                    <host>clickhouse-04</host>
                    <port>9000</port>
                </replica>
            </shard>
        </cluster_3_replicas>        
    </remote_servers>

    <zookeeper>
        <node>
            <host>zookeeper</host>

        </node>
    </zookeeper>

    <macros>
        <shard>01</shard>
        <replica>clickhouse-01</replica>
    </macros>
</clickhouse>
```
